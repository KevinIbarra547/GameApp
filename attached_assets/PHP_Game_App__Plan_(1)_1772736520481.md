# PHP Web Game \- Final Project Overview

## Introduction

You will design and build an **original web-based game** using PHP, HTML, and CSS. You will choose one of two tracks:

* **Text-Based Game** (PHP / HTML / CSS) with data stored in JSON Files  
* **JavaScript-Based Game** (JavaScript / PHP / HTML / CSS) with data stored in JSON Files

Your game must allow players to **save their progress using a name** (a simple string). No login or account system is required. Implementing a login/account system is available as **extra credit**.

### **Project Requirements**

Your game must meet two categories of requirements: gameplay requirements that shape the player's experience, and technical requirements that define how the project is built.

**Gameplay Requirements (what the player experiences):**

1. Score tracking that updates as the player progresses  
2. Game over state with restart capability  
3. At least one visual effect triggered by gameplay (JS-based games may also use sound)  
4. Sortable leaderboard by at least 3 criteria: score, player name, and one additional value you choose  
5. Save gameplay using a name or username (login/account system available as extra credit)

**Technical Requirements (how the project is built):**

1. At least two JSON data files  
2. Reusable PHP functions in `functions.php` (minimum 3\)  
3. An "About" page with game rules, credits, and AI usage documentation  
4. A GitHub repository with at least 12 targeted commits (provided in Phase 2\)

## AI Policy

You are **allowed and encouraged** to use AI tools throughout this project. AI can help you brainstorm, debug, learn new concepts, and write/explain code. However, AI cannot replace your ability to understand and explain your project. All AI usage must be documented in your About page.
